[~] REC-172 — Pipeline GitHub Actions per build container guacenc
------------------------------------------------------------------------------
  Priorita: MEDIA
  Dipendenze: REC-171

  DESCRIZIONE:
  Creare un workflow GitHub Actions standalone dedicato per la build e il push
  del container `guacenc`. Il workflow deve allinearsi agli standard degli altri
  container, includendo la build multi-architettura (linux/amd64, linux/arm64)
  tramite QEMU e Buildx, il versionamento semantico (tag generati
  automaticamente) e il push dell'immagine su ghcr.io.

  DETTAGLI TECNICI:
  - Creare un nuovo workflow dedicato `.github/workflows/guacenc-build.yml`.
  - Configurare i trigger su push/tag, in modo simile a `ssh-gateway-build.yml`.
  - Aggiungere il job `build-and-push` per la compilazione multi-architettura con
    `docker/build-push-action`, usando il contesto `docker/guacenc` e il
    relativo `Dockerfile`.
  - Includere la corretta logica per l'estrazione metadata (tag major/minor/patch).
  - Implementare il job `scan` che esegue `trivy` sull'immagine locale e carica i
    risultati SARIF nella Security tab di GitHub Actions.

  File coinvolti:
    CREARE:     .github/workflows/guacenc-build.yml
